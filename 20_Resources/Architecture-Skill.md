# 对话素材：OpenClaw 技能架构与全局规则配置
**日期**：2026-03-11
**主题**：从本地技能到全局规则的演进路径

---

## 1. 初始状态查询
**用户问**：帮我看看当前的 skills 配置，都安装了哪些 skills？

**过程记录**：
Agent 检查了本地工作区 (`skills/`) 和系统全局目录 (`/home/aatroxli/.npm-global/...`)。
- **本地**：仅 `skill-vetter`。
- **全局**：包含 `clawhub`, `skill-creator`, `feishu-doc`, `weather` 等核心与扩展技能。

---

## 2. 核心概念辨析：全局 vs 本地
**探讨内容**：
- **全局技能**：随系统升级，全员共用，适合通用能力。
- **本地技能**：工作区私有，支持深度定制，适合特定项目。
- **多 Agent 兼容性**：只要 Workspace 路径一致，子 Agent 可以共享本地技能。

---

## 3. 架构演进：从本地迁移到全局
**用户决策**：为了支持“多 Agent + 多 Workspace”的隔离模式，决定将 `skill-vetter` 迁移到全局。

**操作记录**：执行 `mv` 指令将技能从本地 Workspace 移至全局目录。

---

## 4. 规则持久化的探索
**用户问**：指令保存到哪里？重启后还记得吗？

**演进逻辑**：
1. **方案一 (USER.md)**：将规则写入工作区的 `USER.md`。
   - *缺陷*：Workspace 隔离后，新工作区读不到。
2. **方案二 (内嵌 Skills)**：修改现有技能或建立全局技能。
   - *优势*：全局技能在任何 Workspace 都能被加载，逻辑最稳固。

---

## 5. 最终落地：全局规则技能 `global-rules`
**执行过程**：
1. 确定使用 `skill-creator` 的规范化思路。
2. 创建全局技能 `global-rules`。
3. **SKILL.md 核心内容**：
    - 约束 `clawhub install` 的行为：参数不全时必须询问。
    - 确立“多 Workspace 隔离”为最高准则。
4. **清理**：删除 `USER.md` 中的冗余配置，统一指向 `global-rules`。
