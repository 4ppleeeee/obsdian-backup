# 写作素材：PVE + 反向代理环境下的 AI Agent Gateway 部署实战
**日期**：2026-03-11
**主题**：多层级网络架构下的 WebSocket 穿透与安全加固

---

## 1. 架构场景
- **宿主机**：PVE (Proxmox Virtual Environment)
- **Agent 虚拟机**：Ubuntu VM (192.168.31.82)，监听端口 22025。
- **反代虚拟机**：Nginx/Caddy VM (192.168.31.101)，负责 SSL 卸载与公网暴露。

---

## 2. 关键冲突解决：LAN vs Tailscale Funnel
**排查记录**：
- **现象**：修改 `gateway.bind="lan"` 后服务启动失败，报错提示必须绑定到 `loopback`。
- **根源**：`gateway.tailscale.mode="funnel"` 开启时，为了安全，系统强制要求 Gateway 只能监听回环地址（127.0.0.1）。
- **对策**：由于已具备独立反代服务器，决定关闭 Tailscale Funnel (`mode: "off"`)，释放局域网监听权限。

---

## 3. Nginx 反向代理配置要点 (192.168.31.101)
**技术核心**：
- **WebSocket 支持**：必须配置 `Upgrade` 和 `Connection` 请求头。
- **端口映射**：将外部 443 流量映射至内部 `192.168.31.82:22025`。
- **安全加固**：在 OpenClaw 端开启 `password` 认证模式。

---

## 4. 实战反思：AI Agent 的“网络身份”
**探讨内容**：
- AI Agent 不应仅仅是一个本地脚本，它在多 VM 环境下具备独立的网络节点属性。
- **配置的连贯性**：从修改端口到处理系统底层安全策略（Funnel 约束）的逻辑链条，展示了 AI 系统在复杂网络环境下的自我诊断能力。
