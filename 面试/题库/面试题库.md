## 第一阶段：Android 核心与语言特性 (10 题)

### 1. Activity 生命周期与资源回收

- **【切入点】**：简历“Android 客户端基础” + GitHub 项目中“数据看板停留时长统计”。
- **【问题】**：在用户正在聊天时突然接到电话，当前 Activity 的生命周期会发生怎样的切换？如果在 `onStop()` 方法中执行过于耗时的数据库保存操作，会带来哪些风险？
- **【答案】**：状态切换为 `onPause() -> onStop()`。风险在于：1. 阻塞后续 Activity 启动体验；2. 内存紧张时进程可能在 `onStop` 完成前被回收导致数据丢失；3. 易引发 ANR。
- **【延伸方向】**：如何利用 `LifecycleObserver` 实现生命周期感知的组件解耦？
- **【延伸方向答案】**：通过实现 `DefaultLifecycleObserver` 接口，让组件自主感知事件，降低 Activity 代码耦合和内存泄漏风险。

### 2. Fragment 的稳定性设计

- **【切入点】**：GitHub 项目采用多 Fragment 架构（消息、看板、搜索）。
- **【问题】**：为什么官方强烈建议通过 `setArguments` 传参，而不是直接利用带参构造函数？
- **【答案】**：系统重建 Fragment 时会自动调用无参构造函数，并利用 `setArguments` 保存的 Bundle 恢复数据。
- ~~**【延伸方向】**：如何确保复用 Fragment 时 ViewModel 的生命周期范围正确？~~
- ~~**【延伸方向答案】**：采用 `navGraphViewModels` 或接口回调，避免强制转换宿主导致的异常。~~

### 3. Kotlin 延迟加载原理

- **【切入点】**：简历“熟练运用 Vibe Coding”及项目开发习惯。kotlin 是否了解？
- **【问题】**：请解释 `by lazy` 的实现原理。它在多线程下如何保证初始化逻辑只执行一次？
- **【答案】**：默认使用同步锁模式。通过双重检查锁定（DCL）逻辑，首次访问时同步执行并缓存，后续直接返回。
- **【延伸方向】**：如果初始化轻量且允许重复初始化，应如何配置？
- **【延伸方向答案】**：模式设为 `LazyThreadSafetyMode.PUBLICATION`，使用无锁化初始化以追求极致性能。

### 4. 序列化性能优化

- **【切入点】**：GitHub 项目 MessageModel 的本地持久化传输。
- **【问题】**：`Serializable` 和 `Parcelable` 的性能差异原因？ Activity 间传递大数据量模型首选哪种？
- **【答案】**：`Serializable` 使用反射产生大量临时对象，GC 频繁；`Parcelable` 手动读写二进制流，内存效率极高。首选后者。
- **【延伸方向】**：如何利用 Kotlin `kotlinx-parcelize` 简化样板代码？
- **【延伸方向答案】**：添加 `@Parcelize` 注解，由编译器自动生成逻辑，兼顾性能与开发效率。

### 5. Context 内存泄漏防护

- **【切入点】**：简历“Android 客户端基础”及多线程开发背景。
- **【问题】**：`Application Context` 和 `Activity Context` 的区别？哪些场景必须用后者？
- **【答案】**：Application Context 全局存活，Activity Context 随界面销毁。显示对话框、启动有结果要求的 Activity 必须用后者。
- **【延伸方向】**：ContextWrapper 模式如何实现在不改代码的情况下切换多语言？
- **【延伸方向答案】**：通过重写 `attachBaseContext`，动态返回包装了新配置信息的 Context。

### 6. Kotlin 扩展函数边界

- **【切入点】**：GitHub 项目中 Utils 工具类的 Kotlin 实现。
- **【问题】**：扩展函数在字节码层面是如何实现的？它能访问类的私有成员吗？
- **【答案】**：编译为静态方法，接收者对象作为首参传入。不能访问私有成员，因为它本质是类外部的工具方法。
- **【延伸方向】**：为什么扩展属性不能持有真实的状态（backing field）？
- **【延伸方向答案】**：因为它不在类体内，无法动态向类实例添加新字段，只能定义 getter/setter 逻辑。

### 7. Service 运行机制

- **【切入点】**：项目“动态消息中心” Handler 模拟推送逻辑。
- **【问题】**：`startService` 和 `bindService` 生命周期差异？如何实现既能后台运行又能交互？
- **【答案】**：`startService` 需手动停止，`bindService` 随绑定者销毁。应先 start 后 bind。
- **【延伸方向】**：Android 8.0+ 后对后台 Service 的限制及替代方案？
- **【延伸方向答案】**：后台限制严苛。推荐使用 `WorkManager` 或 `ForegroundService` 并在通知栏显示。

### 8. Kotlin 协程调度

- **【切入点】**：简历“掌握 Android 运行原理、多线程并发”。
- **【问题】**：`Dispatchers.Main` 和 `Dispatchers.IO` 的底层线程池有何不同？
- **【答案】**：Main 绑定主线程 Looper；IO 使用弹性线程池，针对磁盘/网络 IO 优化。
- **【延伸方向】**：协程中 `withContext` 和 `async` 的使用选择？
- **【延伸方向答案】**：单一挂起任务切线程用 `withContext`；并发多个任务取结果用 `async` 配合 `await`。

### 9. 注解处理器 (APT) 原理

- **【切入点】**：简历“熟悉架构设计”，可能涉及组件化框架。
- **【问题】**：`ARouter` 这种框架是在编译期还是运行期起作用的？其核心优势是什么？
- **【答案】**：编译期通过 APT 扫描注解生成映射表类。优势在于运行时无反射，启动极快。
- **【延伸方向】**：KSP 相比传统的 KAPT 性能提升在哪里？
- **【延伸方向答案】**：KSP 直接理解 Kotlin 符号，无需生成 Java 桩代码，编译速度提升 2x 以上。

### 10. JVM 与 Android 内存管理

- **【切入点】**：简历“了解 JVM 基本原理”。
- **【问题】**：JVM 的 GC 机制与 Android ART 虚拟机的 GC 有什么主要区别？
- **【答案】**：ART 针对移动端优化，支持移动并发 GC，减少了应用暂停（STW）时间。
- **【延伸方向】**：如何手动触发一次强制 GC 进行内存泄漏排查？
- **【延伸方向答案】**：通过 `System.gc()` 配合 `runFinalization()`，或者在 Profiler 中手动点击。

## 第二阶段：架构设计与数据流 (10 题)

### 1. LiveData 粘性处理

- **【切入点】**：GitHub 项目使用 `LiveData` 构建数据链路。
- **【问题】**：Fragment 切换导致的数据倒灌原因及成熟方案？
- **【答案】**：原因是 LiveData 版本号机制。方案：使用 `SingleLiveEvent` 或封装 `Event` 包装类。
- **【延伸方向】**：Kotlin `SharedFlow` 如何精确控制粘性？
- **【延伸方向答案】**：配置 `replay` 参数，`replay=0` 即可彻底规避粘性。

### 2. 单一数据源原则

- **【切入点】**：GitHub 项目中“Repository 封装 SQLite/网络”。
- **【问题】**：如果数据库和缓存数据不一致，以哪个为准？如何设计同步策略？
- **【答案】**：必须以数据库（SSOT）为准。网络数据 -> 写入 DB -> UI 监听 DB 自动更新。
- **【延伸方向】**：高频写入场景下如何保证不阻塞 UI？
- **【延伸方向答案】**：使用 `Dispatchers.IO` 配合事务，并利用 `DiffUtil` 合并 UI 变更通知。

### 3. MVI 架构优势

- **【切入点】**：简历“熟悉 MVVM/MVP 等架构”。
- **【问题】**：MVI 相比 MVVM 最核心的区别？如何减少状态冲突？
- **【答案】**：MVI 强制单一数据流和唯一状态对象，彻底杜绝状态碎片化。
- **【延伸方向】**：Compose 为何更契合 MVI？
- **【延伸方向答案】**：Compose 本质是 `State -> UI` 的函数式映射，MVI 的不可变状态天然匹配。

### 4. Repository 缓存策略

- **【切入点】**：项目“消息列表展示和聊天记录查询”。
- **【问题】**：如何设计三级缓存（内存、DB、网络）？
- **【答案】**：内存缺失查 DB，DB 返回的同时异步发起网络更新，保证展示速度与数据新鲜。
- **【延伸方向】**：如何封装通用的 `Resource<T>` 类？
- **【延伸方向答案】**：使用 `sealed class` 包含 Loading、Success、Error，统一样板代码处理。

### 5. 模块化与组件化

- **【切入点】**：GitHub 项目中功能模块（message/chat/search）的划分。
- **【问题】**：项目中模块间如何通信且保持解耦？
- **【答案】**：定义公共 `Interface` 模块，通过路由框架或 ServiceLocator 注入实现。
- **【延伸方向】**：组件化后如何处理公共库版本的版本冲突？
- **【延伸方向答案】**：建立统一的 `version.gradle` 或使用 Gradle `Version Catalog` 进行版本管理。

### 6. 数据绑定与解耦

- **【切入点】**：简历“熟悉 Activity、Fragment、Service 等常用组件”。
- **【问题】**：DataBinding 在架构中的作用及优缺点？
- **【答案】**：优点是减少 findViewById、UI 逻辑声明式；缺点是 XML 调试难、编译耗时增加。
- **【延伸方向】**：ViewBinding 相比 DataBinding 的优势？
- **【延伸方向答案】**：更轻量、类型安全、零反射、支持 Java/Kotlin 编译更快。

### 7. 响应式编程与并发

- **【切入点】**：简历“熟悉多线程并发及 Spring Boot”。
- **【问题】**：在 Android 中，RxJava 和协程 Flow 该如何选择？
- **【答案】**：协程 Flow 语法更简洁、天然支持结构化并发，是目前 Android 官推方案。
- **【延伸方向】**：Flow 中的 `collect` 和 `collectLatest` 区别？
- **【延伸方向答案】**：`collectLatest` 会取消前一个未完成的任务处理最新值，适用于搜索联想词场景。

### 8. 错误处理与重试机制

- **【切入点】**：GitHub 项目中“弱网模拟与重试流程”。
- **【问题】**：如何优雅地在 Repository 层实现自动重试逻辑？
- **【答案】**：利用协程的 `retry` 操作符或在 `catch` 块中递归/循环触发，并加入指数退避策略。
- **【延伸方向】**：如何避免重试机制导致服务器雪崩？
- **【延伸方向答案】**：引入熔断机制或最大重试次数限制，并在本地记录重试频次。

### 9. UI 状态持久化

- **【切入点】**：GitHub 项目中“聊天记录本地持久化”。
- **【问题】**：当 Activity 因内存压力被销毁，如何恢复用户正在输入的草稿？
- **【答案】**：在 `onSaveInstanceState` 保存到 Bundle，或利用 `ViewModel` 的 `SavedStateHandle`。
- **【延伸方向】**：SavedStateHandle 的实现原理？
- **【延伸方向答案】**：它是对 `onSaveInstanceState` 的封装，将数据自动存储在 System Server 进程中。

### 10. 领域驱动设计 (DDD) 初探

- **【切入点】**：简历“具备团队协作与版本管理经验”。
- **【问题】**：随着业务复杂度增加，你会如何拆分项目的 Data 层和 Domain 层？
- **【答案】**：Domain 层只包含 UseCase 和实体模型，完全脱离平台框架，方便测试和多端迁移。
- **【延伸方向】**：Domain 层是否可以包含 Android SDK 的引用？
- **【延伸方向答案】**：严禁包含，应保持纯 Kotlin/Java，确保核心业务逻辑的独立性。

## 第三阶段：性能优化与工程实践 (10 题)

### 1. RecyclerView 列表调优

- **【切入点】**：GitHub 项目中“会话列表分页加载”。
- **【问题】**：如何优化大列表的首屏加载性能？
- **【答案】**：利用 `DiffUtil` 异步差分、异步预加载数据、使用 `ConstraintLayout` 减少布局嵌套。
- **【延伸方向】**：`setItemViewCacheSize` 的作用？
- **【延伸方向答案】**：增加离屏缓存，滑回时直接复用，显著减少 `onBindViewHolder` 调用。

### 2. 内存泄漏深度排查

- **【切入点】**：简历“熟悉 Android 客户端开发基础”。
- **【问题】**：除了单例持有 Activity，还有哪些隐蔽的泄漏（如内部类 Handler）？
- **【答案】**：非静态内部类 Handler、未释放的 Sensor 监听、协程未绑定生命周期 Scope。
- **【延伸方向】**：如何使用 LeakCanary 引用链判断是系统代码还是业务代码泄漏？
- **【延伸方向答案】**：观察引用链中“加粗”节点，业务类名（如 `MyActivity`）即为泄露源。

### 3. UI 卡顿与渲染优化

- **【切入点】**：简历“具有大模型项目设计经验”涉及数据看板。
- **【问题】**：如何通过代码实时监控掉帧？
- **【答案】**：利用 `Choreographer` 监控帧间隔。优化：避免 `onDraw` 创建对象、减少过度绘制。
- **【延伸方向】**：谈谈 `Overdraw`（过度绘制）排查手段。
- **【延伸方向答案】**：开发者选项开启“调试 GPU 过度绘制”，通过颜色判断层级压力。

### 4. 线程池并发管理

- **【切入点】**：简历“多线程并发、Java 基础”。
- **【问题】**：项目中如何设计高性能线程池？
- **【答案】**：根据任务类型（IO vs CPU 密集）配置核心线程数，使用 `PriorityQueue` 处理紧急任务。
- **【延伸方向】**：协程 `Dispatchers.IO` 线程数上限？
- **【延伸方向答案】**：默认 64 或 CPU 核心数取大者，动态扩容应对高并发阻塞任务。

### 5. 图片加载与内存占比

- **【切入点】**：GitHub 项目中包含“图片消息展示”。
- **【问题】**：一张 1080p 图片占多少内存？如何采样压缩？
- **【答案】**：约 8MB。通过 `inSampleSize` 采样，根据 ImageView 大小按比例加载。
- **【延伸方向】**：`Bitmap.Config.HARDWARE` 的优势？
- **【延伸方向答案】**：直接存入显存，零拷贝加速渲染，大幅节省堆内存。

### 6. ANR 排查与规避

- **【切入点】**：简历“具有大模型项目调优经验”。
- **【问题】**：主线程 `SharedPreferences.commit()` 的风险？
- **【答案】**：同步写磁盘会导致 UI 阻塞引发 ANR。应改用异步的 `apply()`。
- **【延伸方向】**：如何分析 `traces.txt` 文件？
- **【延伸方向答案】**：搜索 `main` 线程堆栈，查看是否有耗时 IO 或死锁等待。

### 7. APK 瘦身实践

- **【切入点】**：简历“具备团队协作与版本管理经验”。
- **【问题】**：你会从哪些维度压缩包体积？
- **【答案】**：WebP 替代 PNG、混淆代码移除无用资源、`resConfigs` 只留中文、只保留 `v7a` SO 库。
- **【延伸方向】**：R8 相比 ProGuard 的优势？
- **【延伸方向答案】**：更深度的无效代码剥离、更好的编译性能和代码缩减率。

### 8. 弱网稳定性优化

- **【切入点】**：GitHub 项目中“弱网模拟”功能。
- **【问题】**：在弱网环境下，如何保证埋点上报不拖累业务？
- **【答案】**：建立本地缓冲池，进入后台或批量聚合后低优先级异步上报。
- **【延伸方向】**：如何设计消息超时重连逻辑？
- **【延伸方向答案】**：指数退避算法（Exponential Backoff）配合最大尝试次数，避免网络雪崩。

### 9. 启动速度优化 (Cold Start)

- **【切入点】**：项目“抖音简版”优秀营员背景。
- **【问题】**：如何缩短冷启动时间？
- **【答案】**：`IdleHandler` 延迟加载 SDK、多线程预加载首屏数据、去除冗余布局层级。
- **【延伸方向】**：`App Startup` 库的原理？
- **【延伸方向答案】**：共用一个 ContentProvider 初始化多个组件，减少多次初始化的系统开销。

### 10. 内存抖动治理

- **【切入点】**：简历“掌握 JVM 基本原理”。
- **【问题】**：什么是内存抖动？在 RecyclerView 滚动时如何避免？
- **【答案】**：频繁创建短生命周期对象触发频繁 GC。方案：使用对象池复用、避免在滑动监听中分配内存。
- **【延伸方向】**：如何监控内存抖动？
- **【延伸方向答案】**：使用 Android Studio Profiler 的 Memory 查看图形波形，若出现密集锯齿状即为抖动。

## 第四阶段：AI 与 LLM 深度思考 (10 题)

### 1. RAG 知识库索引调优

- **【切入点】**：简历中“江苏省人民医院 RAG 知识库构建”。
- **【问题】**：聊天短句 Embedding 效果差，如何优化检索准确率？
- **【答案】**：会话聚合、添加元数据打标。利用轻量模型对对话预生成摘要后进行向量化检索。
- **【延伸方向】**：端侧离线 Embedding 挑战？
- **【延伸方向答案】**：NPU 算力与内存限制。需通过量化（Quantization）或关键词+向量多路召回补齐。
