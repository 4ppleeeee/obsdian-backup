## 核心知识点：锁对象的归属

### **① 修饰普通方法（实例锁 / 对象锁）**

- **锁住的是：** 当前实例对象 `this`。
    
- **影响范围：** 如果两个线程同时访问**同一个对象**的同步方法，会发生竞争；如果访问的是**不同对象**，则不会发生竞争。
    

### **② 修饰静态方法（类锁）**

- **锁住的是：** 当前类的 `Class` 对象（例如 `User.class`）。
    
- **影响范围：** 无论你创建了多少个实例，全家（该类及其所有实例）只有这一把锁。所有线程访问该类的静态同步方法时，都会发生竞争。


**问：** “线程 A 正在访问实例对象 `obj` 的静态同步方法，线程 B 能否同时访问 `obj` 的普通同步方法？”

- **答案：** **可以。**

**问：** “父类和子类都有静态同步方法，它们共用一把锁吗？”

- **答案：** **不共用。**
    
- **原理：** `Parent.class` 和 `Child.class` 是两个不同的对象。

**问：** “线程 A 访问 `synchronized method()`，线程 B 访问 `synchronized(this){...}`，它们互斥吗？”

- **答案：** **互斥。**
    
- **原理：** 因为它们的锁对象都是 `this`。

“如果我用 `synchronized("string_lock")` 锁住一个字符串常量，会有什么风险？” （答案：字符串常量池会导致全局锁定，产生不可预知的死锁，这是一个非常高级的坑）。
